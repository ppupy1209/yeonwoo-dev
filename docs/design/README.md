# 디자인 결정

## 방향: A+B 하이브리드 (확정 · ADR-003)
- **골격:** 에디토리얼 — 여백·타이포 중심 (Vercel/Linear 톤)
- **강조:** 성과 지표 카드(B) — 히어로 직후 4개 메트릭 스트립
- **위트:** 터미널 스타일(C)은 About 섹션 한 곳에만

## 페이지 구조 (single page + 프로젝트 상세)
1. **Nav** — Work / About / Stack / Ask AI(Phase 2)
2. **Hero** — 타이틀 + 한 줄 소개 + CTA(프로젝트 보기, 이력서 PDF)
3. **지표 스트립** — CPU 90→60 · I/O Wait 18→3 · 300건/s · 분산락 중복 0
4. **Selected work** — 프로젝트 카드(문제 → 해결 → 결과 + 성과 뱃지)
5. **프로젝트 상세** — 아키텍처 다이어그램 포함
6. **About** — 터미널 위트(자소서 톤)
7. **Footer** — 연락처 · GitHub · 이력서

## 톤
- **색:** 뉴트럴 베이스 + info(블루) 액센트. 다크모드 지원.
- **폰트:** Sans 본문, 지표·코드 스니펫은 Mono.
- **아키텍처 이미지:** excalidraw 손그림 → 정제된 시스템 다이어그램으로 교체 (1번 프로젝트 샘플 완료).

## 콘텐츠 인벤토리 (노션 → 이전 대상)

| # | 프로젝트 | 스택 | 핵심 성과 |
|---|----------|------|-----------|
| 1 | 실시간 센서 데이터 전송 시스템 | MQTT · ThreadPool · Locust | CPU 90→60% |
| 2 | 선박 운항 데이터 수집 엔진 | Netty · TCP/Serial · Batch Insert | I/O Wait 18→3% |
| 3 | 국면 전환 자동 종료 기능 | Redis 분산락 · AOP `@ScheduleLock` | 멀티 인스턴스 중복 실행 차단 |
| 4 | 전자 결재 시스템 | 조건부 UPDATE · affected rows | Lost Update 차단 |

각 프로젝트 공통 서술 구조: **개요 → 아키텍처 → 문제 원인 → 해결 과정 → 결과.**
