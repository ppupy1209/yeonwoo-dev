## 개요
멀티 인스턴스 환경에서 각 인스턴스의 스케줄러는 동일 주기로 실행되지만, **분산 락을 획득한 단 하나의 인스턴스만** 국면 종료 로직과 WebSocket 알림을 수행하도록 설계했다.

## 문제
- **기술적 맥락**: 가용성을 위해 멀티 인스턴스로 운영 → 각 인스턴스의 스케줄러가 개별적으로 트리거되는 구조.
- **중복 실행**: 특정 국면이 종료 조건에 도달했을 때 모든 인스턴스가 동시에 종료 로직을 수행할 수 있었다.
- **데이터 정합성**: 동일 레코드 경합이 발생하고, WebSocket 종료 알림이 중복 발송되어 서버 데이터와 실제 UI 상태가 불일치하는 현상 발생.

## 해결
- **Redis 기반 분산 락**: 지정된 주기에 단 하나의 인스턴스만 국면 종료 작업을 수행하도록 분산 락 메커니즘 구축.
- **AOP 기반 스케줄러 락 공통화**: 스케줄러마다 반복되는 락 획득/해제 코드를 커스텀 `@ScheduleLock` 애노테이션 + AOP로 선언적 구조로 모듈화.

## 결과
- 멀티 인스턴스 스케줄러 중복 실행 및 WebSocket 중복 알림 문제 해소
- `@ScheduleLock` + AOP로 락 제어 로직 공통화 → 신규 스케줄러 추가 시 코드 중복 없이 즉시 확장
